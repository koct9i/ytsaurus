# Общие сведения

В этом разделе описана общая модель контроля доступа к таблицам и другим узлам [Кипариса](../../../user-guide/storage/cypress.md), а также к прочим системным объектам: пользователям, группам, аккаунтам, чанкам и транзакциям.

Любой запрос в {{product-name}} проходит две последовательные стадии:

1. **Аутентификация.** Прокси проверяет личность пользователя по токену, переданному в запросе. Как правило, токен выдается через [OAuth](https://{{lang}}.wikipedia.org/wiki/OAuth). Если токен в запросе отсутствует, запрос считается выполненным от имени пользователя `guest`. После аутентификации токен больше не используется на следующих этапах обработки. Подробности приведены в разделе [Аутентификация](../../../user-guide/storage/auth.md).
2. **Авторизация.** Мастер-сервер Кипариса проверяет, разрешено ли аутентифицированному пользователю выполнять требуемое действие над целевым объектом.

Результат авторизации зависит от трёх параметров:

1. Пользователя, инициировавшего запрос.
2. Запрошенного права, например `read` или `write`.
3. Объекта, к которому запрашивается доступ.

Если доступ запрещён, {{product-name}} возвращает ошибку, в которой указаны пользователь, объект и право, из-за которого проверка не прошла.

Если прокси отправляет запрос от имени несуществующего пользователя, мастер-сервер возвращает ошибку `No such user`. Такая ситуация возможна, поскольку выпуск токена и регистрация пользователя выполняются разными сервисами и не являются одной операцией.

## Ключевые понятия

- **Субъект** — пользователь или группа.
- **Объект** — сущность, для которой проверяются права: узел Кипариса, аккаунт, транзакция, операция и так далее.
- **Дескриптор управления доступом** (**ACD**) — набор атрибутов объекта, задающих правила доступа: `@acl`, `@inherit_acl` и `@owner`.
- **Список управления доступом** (**ACL**) — значение, хранящееся в `@acl`.
- **Запись управления доступом** (**ACE**) — отдельное правило внутри ACL.

В этой статье рассматриваются общие ACL. О специализированных механизмах читайте в разделах [Управление доступом к колонкам таблиц](../../../user-guide/storage/columnar-acl.md) и [Построчный контроль доступа](../../../user-guide/storage/row-level-security.md).

## Пользователи и группы { #users_groups }

Реестр субъектов хранится в Кипарисе:

- все пользователи находятся в `//sys/users`, это узел типа `user_map`;
- все группы находятся в `//sys/groups`, это узел типа `group_map`.

{% note warning "Внимание" %}

Пользователи и группы находятся в одном пространстве имён. Имя пользователя и имя группы не могут совпадать.

{% endnote %}

Членство в группах транзитивно. Группа может содержать пользователей и другие группы, при этом циклы запрещены. Поэтому, если пользователь `A` входит в группу `B`, а группа `B` входит в группу `C`, то при авторизации пользователь `A` считается членом группы `C`.

#### Как посмотреть списки пользователей и групп

{% list tabs %}

- Через веб-интерфейс

  - Чтобы просмотреть список пользователей, откройте раздел **Users** на боковой панели или перейдите по адресу `{{cluster-ui}}/users`.
  - Чтобы просмотреть список групп, откройте раздел **Groups** или перейдите по адресу `{{cluster-ui}}/groups`.

- Через CLI

  - Получить список пользователей:
    ```bash
    yt --proxy <cluster-name> list //sys/users
    ```
  - Получить список групп:
    ```bash
    yt --proxy <cluster-name> list //sys/groups
    ```

{% endlist %}

{% note warning %}

Узлы `//sys/users` и `//sys/groups` не являются таблицами, поэтому их нельзя читать через `SELECT`.

{% endnote %}

Пользователи имеют тип системного объекта `user`. Группы имеют тип `group`.

#### Как посмотреть сведения о пользователе или группе

{% list tabs %}

- Через веб-интерфейс

  Откройте страницу пользователя или группы и нажмите значок ![](../../../../images/attrs-icon.png =20x20) справа.

- Через CLI

  - Получить один атрибут пользователя:
    ```bash
    $ yt --proxy <cluster-name> get //sys/users/vasya/@type
    "user"
    ```

  - Получить все атрибуты пользователя:
    ```bash
    $ yt --proxy <cluster-name> get //sys/users/vasya/@
    {
      "id" = "ffffffff-fffffffe-101f5-1407420e";
      "type" = "user";
      "builtin" = %true;
      ...
    }
    ```

  - Получить один атрибут группы:
    ```bash
    $ yt --proxy <cluster-name> get //sys/groups/admins/@type
    "group"
    ```

{% endlist %}
